# HEF Implementation Roadmap

## Phase 1 - Foundation
- EntityId, RoomId, HomeId
- DisplayName and NormalizedName
- Fingerprint
- Domain events

## Phase 2 - Domain
- HomeEntity
- HardwareBinding
- Capability
- Alias
- Lifecycle

## Phase 3 - Registry
- Entity Registry
- Name conflict detection
- Hardware replacement workflow

## Phase 4 - Runtime
- Device Registry integration
- Capability Runtime integration
- Automation Engine integration

## Phase 5 - Intelligence
- Digital Twin integration
- AI learning by entity
- Predictive hardware replacement support

Success criterion: hardware replacement must not require changes to automations, scenes or learned behaviors.
